1. Получить список товаров из products.xml

2. Записать полученные данные в БД.
Для каждого товара записать поля: product_id, title, description, rating, price (если есть inet_price), image
- Если в товаре отсутствует product_id, то это будет означать, что id товара такой же как предыдущий, но товар другой вариации
- В зависимости от рейтинга товара добавить товар в одну из категорий:
	плохой (рейтинг <= 3)
	хороший ( 3 < рейтинг <= 4)
	отличный ( 4 < рейтинг <= 5)

3. Вывести данные

Сделать таблицу с отображением собранных данных

4. По клику на картинку или название товара - переходим на страницу товара.
На этой странице выводится вся имеющаяся информация по товару, форма для написания отзыва, список отзывов.

Форма для написания отзыва включает: Имя, email, текст.
Сделать валидацию вводимых данных на клиенте и на сервере:
Имя - только буквы a-z,A-Z
Email - только адреса электронной почты.
Все поля обязательны для заполнения.
Отзыв отправляется без перезагрузки страницы (записывается в бд) и сразу же появляется в списке отзывов.

Список отзывов выводится в порядке "последние сверху".
В списке выводятся: имя, email, дата, текст.

Рекомендуется реализовать на фреймворке Yii 2.0 или Symfony


Бонус:

- Фоновое выполнение парсинга данных по расписанию.
- Деплой скрипт на сервер, результат на сервере.
- Полнотекстовый поиск по названию и описанию товара.


Результат выложить на github (bitbucket) или прислать архивом с комментариями.
